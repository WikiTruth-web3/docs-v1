产品介绍

Evidence Market 是一个由两套逻辑互补的市场组成的双引擎情报网络。

为了适应不同场景下的信息流转需求，平台构建了两个独立的子产品板块：Evidence Blind Box (证据盲盒) 与 Bounty Token (悬赏令牌)。

1. Evidence Blind Box (证据盲盒)​

“先有证据，后找买家”的被动披露网络

Evidence Blind Box 是 Evidence Market 的基石产品。它主要解决的问题是：当吹哨人已经掌握了高价值或高风险的敏感机密时，如何安全地将其变现，并确保真相最终公之于众？

核心运作逻辑：​

  • 加密铸造：吹哨人将证据通过 TEE 加密并铸造为链上盲盒（Evidence Blind Box），标出买断“底价”与首期“延迟费”。
  • 买家博弈：利益相关方（如犯罪分子）支付底价获取解密密钥。如果他们希望这份证据继续保持私密，就必须持续支付指数级增长的延迟披露溢价 (Delay Premium)。
  • 结局趋向公开：当买家停止支付或无法承担高昂的延迟费时，智能合约将自动释放密钥，真相将向全社会公开。

适用典型场景：​

  • 内部员工掌握企业财务造假证据。
  • 知情人士曝光环境污染或用户数据泄露事件。
  • 匿名黑客获取了某非法组织的犯罪记录。
  • 吹哨人掌握了某非法组织的犯罪记录。
  • 有良知的公务员掌握了腐败官员的犯罪记录。

2. Bounty Token (悬赏令牌)​

“先有需求，后找线索”的主动情报网络

如果说 Evidence Blind Box 解决的是“我有秘密”，那么 Bounty Token 解决的就是“我需要线索”。它将传统的线下悬赏通告、线人制度，升级为了流程透明、匿名隐私、无需信任的去中心化情报需求池。

核心运作逻辑：​

  • 锁定赏金：机构或个人发布特定的信息收集需求，并预先在智能合约中锁定一定数量的 EMC、BTC、USDT 作为赏金。
  • 匿名提交：知情人士通过Bounty Token提交相关线索，整个过程链上隐身，确保人身安全。
  • 验证与发放：悬赏发布者对线索进行评估与核实。一旦验证通过，将释放赏金给情报提供者。

适用典型场景：​

  • 遭遇黑客攻击的 Web3 项目悬赏追回被盗资金的线索。
  • 执法机构或安全公司向全球征集在逃嫌疑人的情报。
  • 受害者维权群体联合悬赏欺诈团伙的内部关键名单。
  • 商业调研机构公开悬赏某上市公司的特定业务数据。
  • 匿名线人提交犯罪活动情报，获取赏金。

总结：双引擎的互补效应​

  • Evidence Blind Box 创造了**“信息盲盒的博弈池”**，利用经济压力倒逼作恶方支付代价，为随机发生的突发曝光提供变现与保护渠道。
  • Bounty Token 创造了**“清晰的信息需求池”**，用预置的真金白银驱动知情人打破沉默，为长期、明确的调查提供情报来源。

这两个市场相互独立却又在底层代币机制上紧密融合，共同构成了 Evidence Market 完整的情报流通基础设施。
